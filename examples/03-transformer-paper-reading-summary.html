论文概览

论文背景与核心贡献

"Attention Is All You Need" 发表于 NIPS 2017（现 NeurIPS），由 Google 团队的 8 位共同一作完成：Ashish Vaswani, Noam Shazeer, Niki Parmar, Jakob Uszkoreit, Llion Jones, Aidan N. Gomez, Lukasz Kaiser, Illia Polosukhin。这篇论文提出了 Transformer 模型——一个纯基于 Attention 机制的序列转录模型，完全不依赖 RNN 或 CNN。

论文的核心贡献在于：证明了仅使用 Attention 机制就能构建强大的序列模型。Transformer 具有网络结构简单、并行度更高、训练速度更快的优势，在机器翻译任务上取得了突破性的结果。

标题解读

"Attention Is All You Need" 直译为"注意力就是你所需要的一切"。李沐评价道："这是个好名字，跟变形金刚（Transformers）表达相同，一语双关。" 这个标题简洁有力地传达了论文的核心主张：抛弃 RNN 和 CNN，Attention 就是唯一需要的组件。

这是个好名字，跟变形金刚表达相同。

-- 李沐评论文标题

实验结果与意义

在 WMT 2014 机器翻译任务上，Transformer 取得了令人瞩目的成绩：

  • 1
    英德翻译 BLEU 28.4
    比之前的 SOTA（State of the Art）高出 2 个 BLEU 分，在当时是一个显著的提升。
  • 2

    在更大规模的数据集上同样取得了优异表现，刷新了当时的最佳纪录。

    仅需 8 块 P100 GPU，训练 3.5 天即可完成。相比之下，之前的模型通常需要数周甚至数月的训练时间。

RNN的困境与Attention崛起

RNN的两大缺点

在 Transformer 出现之前，RNN（及其变体 LSTM、GRU）是序列建模的主流方法。论文的导言部分清晰地指出了 RNN 的两大根本性缺陷：

缺点一：难以并行

RNN 的本质是顺序计算——第 t 步的计算依赖第 t-1 步的输出。这种时序依赖构成了一个无法避免的基本约束，导致 GPU 的并行计算能力无法被充分利用。

缺点二：长序列内存限制

在处理长序列时，RNN 需要在时间步之间传递和保存隐藏状态。随着序列长度增加，内存消耗线性增长，同时对长距离依赖的建模也变得困难。

Attention的引入

Attention 机制在 Transformer 之前已经被广泛应用于序列模型中，但此前它总是与 RNN 结合使用——RNN 负责序列建模，Attention 负责在序列中选择性聚焦。Transformer 的革命性在于：完全抛弃了循环（Recurrent）和卷积（Convolution）结构，纯靠 Attention 完成序列建模任务。

核心思想

论文的核心主张非常激进：不仅用 Attention 联接 Encoder 和 Decoder，连 Encoder 和 Decoder 内部也完全使用 Self-Attention 来替代 RNN。这种设计使得序列中任意两个位置之间的信息传递距离变为 O(1)，彻底消除了 RNN 的顺序计算瓶颈。

模型架构总览

Encoder-Decoder框架

Transformer 采用经典的 Encoder-Decoder 框架。Encoder 将长度为 n 的输入序列映射为长度为 n 的连续表示（Representation）；Decoder 以自回归（Autoregressive）的方式，逐步生成长度为 m 的输出序列——在第 t 步，Decoder 只能看到前 t-1 步的输出。

图：Transformer 完整架构 -- 左 Encoder 右 Decoder，Cross-Attention 连接两者

上图中，左侧 Encoder 将输入编码为连续表示，右侧 Decoder 基于 Encoder 的输出生成目标序列。Encoder 和 Decoder 之间通过 Cross-Attention（交叉注意力）联接，使得 Decoder 在每一步生成时都能"看到"输入序列的全部信息。

超参数设计

Transformer 的超参数设计非常简洁，核心只有两个：N（层数）和 d_model（模型维度）。基线模型中 N=6，d_model=512。所有子层的输出维度统一为 d_model，这样设计是为了方便残差连接（Residual Connection）的加法操作。

设计哲学

所有层的输出维度统一为 d_model = 512。李沐指出："超参数就两个：N 和 d。这使得模型架构非常规整，每一层都可以用相同的结构堆叠。"

编码器：多头自注意力与LayerNorm

Encoder结构

Encoder 由 N=6 个相同的层堆叠而成。每一层包含两个子层（Sub-layer）：第一个是 Multi-Head Self-Attention，第二个是 Position-wise Feed-Forward Network（本质是一个 MLP）。每个子层都使用了残差连接（Residual Connection）和 Layer Normalization，公式为：

output = LayerNorm( x + Sublayer(x) )

这里的 Sublayer 可以是 Multi-Head Self-Attention 或 FFN。残差连接保证了梯度能够顺畅回传，LayerNorm 则稳定了训练过程。

BatchNorm vs LayerNorm

LayerNorm 和 BatchNorm 都是归一化方法，但作用方向不同。这一区别对于序列数据尤为关键：

Batch Normalization

归一化方向：按特征维度（列）归一化，即对同一个特征在 batch 内的所有样本计算均值和方差。

问题：序列长度变化时，不同位置的均值和方差波动很大，导致训练不稳定。因此 BN 不适合序列模型。

Layer Normalization

归一化方向：按样本维度（行）归一化，即对同一个样本的所有特征计算均值和方差。

优势：不受 batch size 和序列长度变化的影响，每个样本独立归一化，训练更加稳定。

图：BatchNorm 按列（蓝色）vs LayerNorm 按行（橙色）归一化方向对比
深度解析：残差连接（Residual Connection）的作用

Transformer 中每个子层的输出都经过残差连接和 LayerNorm：

output = LayerNorm( x + Sublayer(x) )

解决梯度消失：在反向传播时，梯度可以通过残差连接的"短路"路径直接回传到前面的层，避免了梯度在多层之间逐渐衰减的问题。即使中间的 Sublayer 梯度很小，残差路径也能保证梯度的有效传播。

信息保留：残差连接保证原始输入信息 x 直接传递到输出。即使 Sublayer 学不到有用的信息，原始信息也不会丢失——最差情况下，Sublayer 的输出接近 0，输出约等于原始输入。

李沐的观点："残差连接让每一层'至少不比输入差'。" 这种设计使得模型可以通过堆叠更多的层来增加容量，而不必担心层数过多导致训练困难。这也是 Transformer 能够成功堆叠 6 层甚至更深的关键。

一个好用的东西，后来大家的理解和原文可能不一样。

-- 李沐谈 LayerNorm

李沐指出，LayerNorm 后来在各种模型中被广泛使用，但人们对其原理的理解可能已经偏离了原始论文的表述。在实践中，LN 已经成为 Transformer 架构的标准组件。

注意力机制详解

Scaled Dot-Product Attention

Scaled Dot-Product Attention 是 Transformer 中最基础的注意力计算方式。其核心公式为：

Attention(Q, K, V) = softmax( Q · KT / √dk ) · V

计算过程分为四步：首先将 Query（Q）与 Key（K）做内积计算相似度；然后除以 √dk 进行缩放（Scaling）；接着通过 Softmax 转换为权重分布；最后用权重对 Value（V）进行加权求和。

图：Scaled Dot-Product Attention 计算流程 -- Q*K^T -> Scale -> Softmax -> 加权V
为什么要除以 √dk？

当维度 dk 较大时，Q 和 K 的内积结果在数值上会呈现较大的范围。这导致 Softmax 的输出趋近于 one-hot 分布——某个位置接近 1，其余接近 0。这种情况下梯度非常小，训练难以进行。除以 √dk 可以将内积值缩放到合理范围。在 Transformer 中 d=512，Scaling 是必要的。

深度解析：Self-Attention 计算的具体数值示例

以下用一个 4 个 token、2 维的简化示例，逐步展示 Self-Attention 的完整计算过程。

第 1 步：输入矩阵 X（4x2）

X = [[1, 0], [0, 1], [1, 1], [0, 0]]

第 2 步：线性投影得到 Q, K, V（假设 W_Q = W_K = W_V = I，便于理解）

Q = K = V = X = [[1, 0], [0, 1], [1, 1], [0, 0]]

第 3 步：计算注意力得分 QK^T / sqrt(2)（sqrt(2) 约等于 1.414）

QK^T = [[1, 0, 1, 0], [0, 1, 1, 0], [1, 1, 2, 0], [0, 0, 0, 0]] Scores = QK^T / 1.414 = [[0.71, 0, 0.71, 0], [0, 0.71, 0.71, 0], [0.71, 0.71, 1.41, 0], [0, 0, 0, 0]]

第 4 步：Softmax 归一化（以第 3 行为例）

第3行 softmax([0.71, 0.71, 1.41, 0]): e^0.71 约等于 2.03, e^1.41 约等于 4.10, e^0 = 1 总和 = 2.03 + 2.03 + 4.10 + 1.00 = 9.16 权重 约等于 [0.22, 0.22, 0.45, 0.11]

第 5 步：加权求和得到输出 Output = Weights x V

第1行: 0.34*[1,0] + 0.14*[0,1] + 0.34*[1,1] + 0.17*[0,0] = [0.68, 0.48] Output 约等于 [[0.68, 0.48], [0.48, 0.68], [0.67, 0.67], [0.50, 0.50]]

可以看到：第 3 个 token [1,1] 对自身的注意力权重最高（0.45），因为它的"信息量最大"。第 4 个 token [0,0] 的权重分布最均匀（各 0.25），因为它不包含任何信息来区分与其他 token 的相似性。

深度解析：为什么需要 sqrt(dk) 缩放（数学推导）

前提假设：假设 q 和 k 的每个分量独立、均值为 0、方差为 1。

推导过程：

q · k^T = SUM(i=1..dk) qi · ki

这是 dk 个独立随机变量乘积之和。每个乘积 qi · ki 的均值为 0，方差为 1（因为 E[qi*ki] = E[qi]*E[ki] = 0，Var[qi*ki] = E[qi^2]*E[ki^2] = 1）。

Var(q · k^T) = dk · Var(qi · ki) = dk

当 dk 很大时（如 dk = 512），q · kT 的值域会非常大（标准差为 sqrt(512) 约等于 22.6）。

问题所在：Softmax 函数在输入值很大时，输出趋近于 one-hot 分布。此时梯度几乎为 0，导致梯度消失，训练无法有效进行。

解决方案：除以 sqrt(dk)

Var(q · k^T / sqrt(dk)) = dk / dk = 1

缩放后，方差恢复为 1，Softmax 的输入值保持在合理范围，梯度可以正常传播。这就是为什么公式中需要除以 sqrt(dk)。

此外，在 Decoder 中还使用了 Mask 机制：将不该看到的位置的 K 值设为极大负数，经过 Softmax 后这些位置的权重变为 0，从而实现对未来信息的遮蔽。

Multi-Head Attention

Multi-Head Attention 的核心思想是：不直接在 d_model 维度上做单次 Attention，而是先将 Q、K、V 分别通过线性投影到较低维度，并行做 h 次独立的 Attention，然后将结果拼接（Concat）后再次线性投影回 d_model 维度。

MultiHead(Q, K, V) = Concat(head1, ..., headh) · WO
其中 headi = Attention(Q · WiQ, K · WiK, V · WiV)

基线模型中 h=8 个头，每头维度为 dk = dv = d_model / h = 64。这种设计借鉴了 CNN 中多通道（Multi-Channel）的思想——不同的头可以学习关注输入的不同"视角"和模式。

图：Multi-Head Attention -- Q,K,V 投影到 h 个头，并行 Attention 后 Concat + Linear

单一点积注意力无可学习参数，投影让模型学习不同"视角"的相似函数。

-- 李沐解释 Multi-Head 的动机

三种应用场景

Attention 机制在 Transformer 中有三种不同的应用方式，分别服务于不同的目的：

1
Encoder Self-Attention（编码器自注意力）

Q = K = V = Encoder 输入本身。每个位置都可以直接看到序列中的所有其他位置。信息传递距离为 O(1)，这是 Transformer 相比 RNN 的核心优势。

2
Decoder Masked Self-Attention（解码器掩码自注意力）

Q = K = V = Decoder 已输出的内容，但加入了 Mask 机制。在第 t 步，只能看到位置 1 到 t-1 的信息，保证训练和预测的一致性——预测时确实无法看到未来的输出。

3
Encoder-Decoder Cross-Attention（交叉注意力）

Q 来自 Decoder 上一层的输出，K 和 V 来自 Encoder 的最终输出。这是 Encoder 和 Decoder 之间的信息桥梁：Decoder 在生成每个词时，可以通过 Attention 直接"查询"输入序列中与当前生成最相关的部分。

解码器、位置编码与前馈网络

Decoder与Mask机制

Decoder 同样由 N=6 个相同的层堆叠。每层包含三个子层：Masked Multi-Head Self-Attention、Multi-Head Cross-Attention（接收 Encoder 输出）和 Position-wise FFN。每个子层同样使用残差连接 + LayerNorm。

Mask机制详解

Mask 机制确保在训练时，第 t 个位置只能看到 1 到 t-1 位置的信息。实现方式是将 t 之后位置的注意力得分设为极大负数（如 -1e9），经过 Softmax 后这些位置的权重变为 0。这保证了训练时的行为与推理时一致——推理时确实无法看到未来的输出。

Positional Encoding

Self-Attention 本身不包含任何序列顺序信息——打乱输入序列的顺序，输出结果不变（除了对应的顺序也变了）。为了让模型感知序列中元素的位置，Transformer 引入了 Positional Encoding。

位置编码使用 sin 和 cos 函数生成：

PE(pos, 2i) = sin( pos / 100002i/d )
PE(pos, 2i+1) = cos( pos / 100002i/d )

其中 pos 是位置索引，i 是维度索引。Positional Encoding 直接加到 Input Embedding 上，使得模型在每一层都能感知到位置信息。选择 sin/cos 函数的好处是：对于任意固定偏移 k，PE(pos+k) 可以表示为 PE(pos) 的线性函数，这使得模型能够学习相对位置关系。

图：Positional Encoding 不同维度的 sin/cos 波形 —— 低频（橙色）捕捉全局位置，高频（蓝色）捕捉局部差异

Position-wise FFN

Position-wise Feed-Forward Network 这个名字听起来复杂，但本质上就是一个 MLP。所谓"Position-wise"是指对序列中的每个位置独立地、以相同的方式应用这个 MLP。

FFN(x) = max(0, xW1 + b1)W2 + b2

维度变化为：512 → 2048 → 512。中间层将维度扩大 4 倍后再压缩回来，ReLU 作为激活函数。这个 MLP 只作用于最后一个维度（特征维度），不同位置之间没有信息交互——序列信息的整合完全由 Attention 层完成。

Transformer 和 RNN 的本质区别在于如何使用序列信息。RNN 通过逐步传递隐藏状态来聚合序列信息，而 Transformer 通过 Self-Attention 一步到位地完成全局信息聚合，然后用 MLP 做非线性变换。

-- 李沐对比 Transformer 和 RNN

Embedding与Softmax

Transformer 中有三处使用了 Embedding 权重，并且它们共享参数（Weight Tying）：Encoder 输入 Embedding、Decoder 输入 Embedding、以及 Decoder 输出层之前的 Linear 变换。此外，Embedding 的输出会乘以 √d_model，目的是使 Embedding 的数值尺度与 Positional Encoding 一致。

Self-Attention优势分析与训练细节

复杂度对比分析

论文中提供了一个关键的对比表格，从三个维度对比了 Self-Attention、RNN 和 CNN：

指标 Self-Attention RNN CNN
每层计算复杂度 O(n2 · d) O(n · d2) O(k · n · d2)
最大信息路径长度 O(1) O(n) O(logk(n))
最小序列操作数 O(1) O(n) O(1)

从表格可以看出，Self-Attention 在信息传递距离上具有绝对优势——序列中任意两个位置之间的信息传递只需 O(1) 步，而 RNN 需要 O(n) 步逐步传递。当序列长度 n 小于表示维度 d 时（大多数实际场景中 d=512，n 通常在 500 以内），Self-Attention 的计算效率也优于 RNN。

归纳偏置（Inductive Bias）

李沐提出了一个关键观点：Self-Attention 的归纳偏置更少、更一般化。RNN 天然假设序列具有时序性，CNN 天然假设局部相关性。而 Self-Attention 不做这些假设，因此需要更多的数据和更大的模型规模才能训练出好的效果。但这也赋予了它更强的通用性和更大的上限。

深度解析：归纳偏置（Inductive Bias）深度分析

定义：归纳偏置是指模型在学习过程中对数据做出的先验假设。这些假设帮助模型在海量可能的解空间中，找到更合理的解。不同的架构有不同的归纳偏置，这决定了它们适合什么类型的任务。

RNN 的归纳偏置：

① 时序性——假设序列中相邻元素的相关性强，距离越远的元素相关性越弱。② 因果性——假设过去的信息影响未来，但不反过来。这使得 RNN 天然适合时间序列、语言建模等任务，但也限制了它捕捉长距离依赖的能力。

CNN 的归纳偏置：

① 局部性——假设邻近元素之间的相关性最高，通过局部卷积核提取特征。② 平移不变性——假设相同的模式出现在序列的任何位置都应被同等对待。这使得 CNN 在图像和局部模式识别中非常高效。

Self-Attention 的归纳偏置：几乎无。

Self-Attention 不假设任何元素之间的优先关系——所有位置之间的连接权重完全由数据学习。它没有局部性假设（所有位置全连接），没有时序假设（位置信息通过 Positional Encoding 额外注入，而非架构内建）。

代价：需要更多数据和更大模型规模才能学到有效模式。因为模型从零开始学习所有关系，没有"先天的知识"帮助它。

收益：上限更高、更通用。一旦数据充足，Self-Attention 可以学到任何模式——无论是局部的还是全局的，时序的还是非时序的。这就是为什么 Transformer 能在 NLP、CV、语音、多模态等几乎所有领域都取得成功。

"Attention is all you need" 标题不完全准确，MLP、残差连接等也缺一不可。

-- 李沐的修正观点

    采用 Warmup 策略：学习率先线性增长到峰值，然后按步数的倒数平方根衰减。公式为 lrate = dmodel-0.5 · min(step-0.5, step · warmup-1.5)，warmup 步数为 4000。
  • 4
    Residual Dropout
    在每个子层的输出（残差连接之前）和 Embedding + Positional Encoding 的和上施加 Dropout，rate = 0.1。基线模型训练 100K 步。
  • 5
    Label Smoothing
    使用 ε = 0.1 的 Label Smoothing。这会损害 Perplexity（困惑度），但能提高 BLEU 分数——因为模型学到的分布更加"不确定"，从而提高了泛化能力。
深度解析：Warmup 学习率策略的直觉

为什么需要 warmup：模型初始参数是随机初始化的，此时模型对数据的理解几乎为零。如果一开始就用很大的学习率，梯度方向会非常不稳定——因为随机参数计算出的梯度方向大概率是错误的。大的学习率配合错误的梯度方向，会导致参数被"甩飞"，后续训练很难恢复。

Warmup 阶段（前 4000 步）：学习率从 0 线性增加到峰值。这就像"开车先热身再加速"——先用小的学习率让参数逐步进入一个合理的区域，等梯度方向稳定后再加速。

Decay 阶段（4000 步之后）：学习率按 step-0.5 衰减。此时模型已经在合理的参数空间中，需要越来越小的步长来精细调整，逐步收敛到最优解。

类比：开车时先慢慢起步（warmup），等引擎热了、方向对了再加速（peak），最后接近目的地时减速（decay），而不是起步就全油门。

深度解析：Label Smoothing 的原理

传统做法（Hard Label）：正确类别的概率设为 1，其余所有类别设为 0。这等价于告诉模型"我 100% 确定这就是答案"。问题在于，模型为了使交叉熵损失最小化，会倾向于输出极其自信的预测概率（如 0.9999...），这容易导致过拟合。

Label Smoothing（Soft Label）：将正确类别的目标概率从 1 降为 1 - epsilon，其余类别均匀分配 epsilon / (V - 1) 的概率（V 是词表大小）。例如 epsilon = 0.1，V = 37000 时，正确类别目标为 0.9，其余每个类别约 0.0000027。

效果：模型不会对任何预测过度自信，输出的概率分布更加"平滑"。这种"不那么确定"的态度反而帮助模型泛化——因为在测试数据中，很少有情况是某个答案 100% 正确的。

Trade-off：Perplexity（困惑度）会变差，因为模型的预测更加"不确定"了（概率分散到了更多词上）。但 BLEU 分数会提升，因为模型在生成时不会因为过度自信而选择不合适的词。这是一个典型的"校准 vs 准确性"权衡。

超参数实验主要调节三个维度：N（层数，2/4/6/8）、d_model（模型维度，128/256/512/1024）、h（头数，1/2/4/8/16/32）。实验结果表明，头数 h=8 是一个较好的平衡点，而增加层数和模型维度通常能带来更好的性能。

总结

Transformer 的核心贡献不仅仅是"用 Attention 替代 RNN"，更重要的是它展示了一种全新的序列建模范式：通过 Self-Attention 实现全局信息聚合，用更少的归纳偏置换取更大的模型容量。这种设计理念催生了 BERT、GPT 系列等划时代的工作，开启了大规模预训练模型的新纪元。正如李沐所言，Transformer 的意义不仅在于这篇论文本身，更在于它所带动的整个研究方向的变革。
